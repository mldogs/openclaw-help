# OpenClaw Help

Практичный репозиторий с нормальной документацией по настройке OpenClaw без лишней магии, шаманства и самообмана.

Сейчас внутри лежит подробная инструкция по интеграции **Langfuse + OpenClaw** с реальными подводными камнями, которые обычно и съедают время.

## Что внутри

- `docs/langfuse-openclaw-integration.md` — подробный пошаговый документ:
  - как включить stock `diagnostics-otel`
  - почему этого недостаточно для content traces
  - как подключить отдельный content-level plugin
  - как проверять результат через API, а не по ощущениям
  - какие утечки и грабли нужно вырезать сразу

## Для кого это

Для тех, кто хочет:
- быстро поднять observability в OpenClaw
- получить в Langfuse не только технические спаны, но и нормальные chat traces
- не утопить всё в system prompt, history и мусорных payload'ах

## Что важно

Этот репозиторий сознательно без секретов:
- без токенов
- без ключей
- без приватных endpoint-specific значений
- без чувствительных runtime-данных

Все примеры здесь либо шаблонные, либо безопасно отредактированы.

## Быстрый старт

1. Открой `docs/langfuse-openclaw-integration.md`
2. Пройди шаги по порядку
3. Проверяй успех через Langfuse API, а не только по локальным логам
4. Не включай в export лишний внутренний контекст, если не хочешь потом разгребать слив prompt-мяса

## Почему этот репозиторий полезен

Потому что типичный путь выглядит так:
- включил tracing
- увидел пустоту
- решил, что всё сломано
- полез дебажить не то место

А правильный путь такой:
- разделить **техническую observability** и **content tracing**
- проверить, что именно реально прилетает в Langfuse
- ужать payload до вменяемого размера
- не слить hidden reasoning и внутренний runtime-контекст

## Статус

Док основан на реальной рабочей настройке, а не на теории.

Если OpenClaw или Langfuse потом поменяют API/хуки, документ всё равно остаётся хорошей отправной точкой: он описывает не только шаги, но и логику проверки.
